Welcome — what this site is

A small place to keep notes on things I'm learning, and the visualizations I build to understand them.

I learn faster when I write things down. I learn even faster when I draw them. This site is where both of those habits live — short notes about whatever I'm currently working through, and the occasional interactive diagram for when a static page isn't enough.

How it's organized

The home page has two sections. Notes are short writings, like this one. Visualizations are self-contained HTML pages that explain something visually — usually because I couldn't find an explanation that clicked for me, so I made one.

Adding new content

To add a note, I create a new .html file in posts/ (using posts/welcome.html as a template) and add an entry to assets/js/posts.js. Same idea for visualizations — drop them in visualizations/ and add a manifest entry.

Why static?

No build step means no maintenance creep. GitHub Pages serves the files directly, and I can preview locally just by opening the HTML in a browser. The tradeoff is more boilerplate per page — that's fine for the volume I'm planning.

The point isn't to publish — it's to remember.
